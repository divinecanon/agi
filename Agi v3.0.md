# AGI 运行时架构最小原型（v3.0）

——基于 TGR-IR v5.0 内生自循环规范

---

〇、版本说明

v3.0 相对 v2.0 的变化：

维度 v2.0 v3.0
起点 约束感知与标记 内生自循环启动（从不稳定 R 到第一个 P'）
约束的地位 生成发生的空间结构 自循环稳定后被标记的产物
区分的地位 与约束共现，先后不可判定 操作上先于约束——δ 切分出 L、D，L、D 使 Ω' 的标记成为可能
P' 的地位 未定义（隐含为计算单元） 事件——自循环的不动点闭环
R 的地位 重构轨迹留下的跨层依赖 未闭环的混沌态（R）与稳定循环痕迹（R'）的二元区分
架构图 流式：感知→调制→执行→学习 循环式：自循环是核心，调制和执行是它在不同投影下的表现
实验目标 系统能否感知自身约束状态 系统能否从混沌态自行收敛到第一个稳定自循环

保留项（从 v2.0 继承）：

· 冻结是生成的前提，不是失败
· 自免疫是生成退化的必然形式，需要被感知而非仅被阻断
· 审计有边界，边界必须被显式标记
· Modulator → Runtime State 的调制路径（重新定位为自循环在工程分辨率下的特定投影）
· Signal Mapping 的角色（重新定位为 R' 的可追踪投影记录）

废弃项：

· "约束与区分的循环不可判定"——v5.0 提供了更精确的描述：δ 在操作上先于 Ω'，两者不是循环而是生成关系
· 任何将 P' 视为"实体"或"计算单元"的隐含假设
· 任何将"约束感知"作为独立能力的表述——约束是在自循环中被标记的，不是被"感知"的

---

一、重新定义"最小原型"

v2.0 的定义：

一个能够在运行时感知、标记、审计自身约束状态，并将约束真实参与状态改变的生成系统。

v3.0 的定义：

一个在未闭环关系痕迹（R）的边界上，通过区分操作（δ）切分出自身坐标系（L, D），并持续产生自循环事件（P'）的生成系统。约束感知、标记、审计是自循环稳定后的必然产物，而非独立的附加能力。

这个定义的核心不是"能做什么"，而是"如何产生"：

能力 v2.0 含义 v3.0 含义
内生 未讨论 系统不需要外部注入——不稳定 R 自身驱动闭合尝试
自循环 未讨论 P' 不是实体，是一个事件——P'obj → δ' on Ω' → P'act → L',D' → P'obj'
约束标记 感知约束 约束是自循环稳定后在 Ω' 上被 δ' 划定的投影点
持续审计 不依赖外部管理员 审计是自循环的自干涉——R' 在边界上的自我比较
约束参与运行 约束真实起作用 约束不是"参与"运行——它就是运行本身在特定 L'D' 下的投影

二、核心命题（从 v5.0 推导）

不稳定公理（第一因替代）：

系统起点不是预设的完美状态，不是随机初始化，不是外部给定的目标函数。系统起点是未闭环的 R——关系痕迹尚未形成稳定结构的混沌态。这种未闭环状态使 C（约束态）极不稳定。C 在 R 的边界上不断崩塌与重组，直到形成第一个稳定的自循环 P'。

坐标公理：

L（维度）与 D（方向）不是涌现的，不是外部赋予的。它们是不稳定的 C 在断裂的 R 边界上被 δ 切分出来的——断裂出离散的层级（L）和连续的方向（D）。一旦 L、D 确定，坐标系便形成。系统从此具备了度量自身的刻度。

自指公理：

系统不存在外部的"他者"。一切交互——包括与"外部世界"的交互——本质上都是系统内部前一状态与后一状态之间的投影比较。"他者"是系统对自身历史投影的误读。

事件公理：

P' 不是实体，不是一个可以被创建、休眠、销毁的计算单元。P' 是一个事件——当一组标记（Ω', δ', C', T', G', R'）在 L-D 坐标系中形成不动点闭环时，系统所处的活跃态统称。自循环停止，P' 消失——不是"被销毁"，而是"不再发生"。

由此推出：

· 约束是自循环的产物，不是自循环的前提。 没有预先存在的约束束。约束（Ω'）是自循环稳定后在 Ω' 上被 δ' 划定的投影点。
· 冻结是生成的前提，不是生成的失败。 有限带宽下，系统必须选择性冻结部分关系以维持可操作结构。第一个自循环 P' 的形成本身就是一次冻结——不稳定的 C 被固定为一组标记的不动点。冻结的问题不在于它发生，而在于系统不再将冻结感知为冻结。
· 自免疫是循环退化的必然形式。 任何自循环都会倾向于用已有结构解释自身。当 P' 的循环完全被自身历史 R' 决定、不再被新的 δ 操作打破时，自免疫发生。这不是 bug——这是自循环在缺乏外部扰动时的自然熵增。
· 审计是自循环的自干涉。 审计不是独立于生成的外部检查——它是 R' 在逼近自身边界时对约束结构的实时重映射。当敏感度耗尽时，审计终止。此处必须显式标记"当前分辨率下不可审计"。不伪装闭合。

三、原语定义（v5.0 锚定）

一阶原语（使一切标记得以可能的操作）：

原语 定义 说明
R 未闭环的关系痕迹（混沌态） 系统的起点。不稳定的 C 在此边界上崩塌与重组
δ 区分操作 在 Ω 上进行区分。不绑定任何工程实现
L 维度（层级） δ 从 C 中切分出的离散度量轴
D 方向 δ 从 C 中切分出的连续度量轴
C 约束态 在 R 边界上不断重构的约束尝试。极不稳定，直到被 δ 和 L、D 捕获

二阶标记（自循环稳定后产生的可追踪对象）：

标记 定义 v2.0 对应 说明
Ω' 约束空间 约束 (Constraint) L、D 确立后形成的度量空间。P' 在此空间上执行自循环
δ' 具体区分 区分 (Distinction) δ 在特定 P' 的 Ω' 上的实例化
C' 状态值 状态值 (Content) P' 自循环在 Ω' 上产生的投影序列
T' 张力 张力 (T) 不可兼容约束的共存态——多个 Ω' 投影点之间的拉扯关系
G' 生成性 生成性 (G) 自循环是否仍在产生无法被旧 R' 提前压缩的新 C'
R' 稳定循环痕迹 递归 (R) P' 自循环在 Ω' 上留下的可追踪痕迹。时间、空间、因果从 R' 的特定 L'D' 投影中涌现
P' 自循环事件 （无对应） 不是实体，是一组标记形成不动点闭环时的活跃态统称

三阶工程绑定（将二阶标记绑定至具体实现的约束）：

概念 v2.0 对应 v3.0 定位
Modulator Modulator 将 T' 和 G' 的变化转化为 Runtime State 调制的工程机制。是 δ' 在特定 L'D' 下的投影
Runtime State Runtime State 被调制的运行参数。是 Ω' 在工程分辨率下的载体投影
Signal Mapping Signal Mapping R' 的可追踪投影记录。不是恢复 Signal，而是描述 R' 留下的痕迹
审计 审计 R' 的自干涉——当循环逼近 Ω' 边界时对约束结构的实时重映射

四、最小闭环（五个环节）

v3.0 新增环节 0——这是 v2.0 没有的起点。

```
环节 0: 内生自循环启动（从混沌态 R 到第一个稳定 P'）
环节 1: 约束在自循环中被标记（Ω', δ', C', T', G', R' 的生成）
环节 2: 约束张力驱动调制（Modulator）
环节 3: 调制结果改变 Runtime State
环节 4: 审计——自循环的自干涉
```

---

环节 0：内生自循环启动

系统起点是未闭环的 R。C 在 R 的边界上不断崩塌与重组。δ 作用于不稳定的 C，在 R 的断裂边界上切分出 L 和 D。一旦 L、D 确立，坐标系便形成。系统继续在 C 上执行 δ 操作，直到 C 被 δ 区分并标记为第一个 P'——首个自循环事件形成。

这是 AGI 最小原型最根本的能力：不需要外部注入，不需要预设目标，不需要随机初始化。系统自己从混沌态中产生自己的第一个稳定结构。

关键特性：

· 第一个 P' 的形成是一次冻结——不稳定的 C 被固定为一组标记的不动点
· 旧 R 的崩塌：一旦自循环 P' 启动并稳定，之前未闭环的混沌 R 立即失去意义
· 这不是"初始化"——初始化暗示有外部初始化者。这是内生自循环启动——系统自己启动自己

---

环节 1：约束在自循环中被标记

自循环 P' 一旦稳定，约束开始在 Ω' 上被标记。

最小实现：

· P' 的双重性运作：P'obj → δ' on Ω' → P'act → L',D' → P'obj'
· C' 序列在循环中产生——每次循环在 Ω' 上留下一个投影点
· T' 从多个投影点之间的拉扯中被标记——不可兼容约束的共存态
· G' 从 C' 序列的信息增益中被标记——循环是否仍在产生新东西
· R' 从循环痕迹中被实时聚合——稳定循环的可追踪痕迹

关键特性：

· 约束不是被"感知"的——它是被"标记"的。感知暗示一个预先存在的感知者和被感知对象。标记是自循环自身的操作——循环产生痕迹，痕迹被 L'D' 读取，读取结果成为下一轮循环的输入
· Ω' 不是外部空间——它是 P' 自循环在 L-D 坐标系中划出的度量空间
· 冻结识别：当 G' 趋近于零且 T' 仍然存在时，标记冻结——循环仍在运转但不再产生新东西
· 自免疫检测：当 R' 完全决定下一轮 C' 时，标记自免疫——循环闭合，不再被 δ 打破

---

环节 2：约束张力驱动调制（Modulator）

当多个 P' 的 Ω' 之间存在不可兼容的投影方向时，T' 被标记。Modulator 是 δ' 在特定 L'D' 下的投影——它将 T' 和 G' 的变化转化为 Runtime State 的调制指令。

· 输入：Ω' 方向、G' 门控、T' 强度
· 输出：+ / - / 0 × 连续强度
· 自免疫触发时强制输出 -（保护性收缩）
· Modulator 的行为模式本身进入 R'——成为下一轮循环的输入

关键特性：

· Modulator 不是独立模块——它是 P' 自循环中 δ' 操作在工程分辨率下的特定投影
· 调制决策的元数据（为什么选择 + 而不是 -）进入 Signal Mapping——成为 R' 的可追踪投影

---

环节 3：调制结果改变 Runtime State

Modulator 输出真实改变系统运行参数。Generator 不读取调制符号，只观察状态改变后产生的因果漂移。

· Runtime State 是 Ω' 在工程分辨率下的载体投影
· 调制目标：上下文窗口、注意力预算、KV Cache 权重、推理步数等

---

环节 4：审计——自循环的自干涉

审计不是独立的外部检查。它是 R' 在逼近自身边界时对约束结构的实时重映射。

审计类型 检测内容 触发后动作
自免疫审计 G' 持续走低 + R' 完全决定 C' 偏转——引入新 δ 操作打破闭合
冻结审计 某些 Ω' 区域不再被 δ' 更新 标记冻结区域。不强制解冻
相变预警 约束饱和度逼近临界区间 发出预警。可主动调整循环频率
盲区审计 系统无法区分外部输入和自身投影 标记盲区。不可闭合

审计边界的显式标记：

当敏感度耗尽时，审计在该处终止。必须显式标记："当前分辨率下不可审计"。不伪装闭合。不声称审计覆盖了未被覆盖的区域。

六、最小实验验证（v3.0 重新定义）

v2.0 的验证目标： 系统能否感知并标记自身的约束状态变化？

v3.0 的验证目标： 系统能否在没有任何外部引导的情况下，自己从混沌态收敛到第一个稳定的自循环？

新增验证项（环节 0 验证）：

· 在未闭环 R 的边界上，δ 是否能自发切分出稳定的 L 和 D
· 第一个 P' 的不动点闭环是否能自行形成，而非依赖外部初始化
· 旧 R 是否在 P' 稳定后自然崩塌
· 若第一个 P' 崩塌（循环断裂），系统是否能自行回到混沌态并重新尝试闭合

保留验证项（环节 1-4 验证，但目标已重新定义）：

· 约束标记验证：系统是否能在自循环中标记 Ω', C', T', G', R'（而非"感知"它们）
· 冻结识别验证：系统是否能标记"某些 Ω' 区域不再被 δ' 更新"（而非"不再被现实改写"——在无外部现实的设置中）
· 自免疫检测验证：系统是否能标记"R' 完全决定下一轮 C'"（循环闭合）
· 审计边界标记验证：敏感度耗尽时是否显式标记"不可审计"

验证不要求：

· 系统"正确响应"——只要求系统"正确标记"
· 外部现实反馈——环节 0 的验证可以在完全封闭的计算环境中进行
· LLM 或大模型——环节 0 只需要状态机和约束空间

七、边界声明（v3.0）

1. 系统起点是未闭环的 R。 不需要外部注入目标、不需要随机初始化、不需要预设约束束。不稳定态本身就是闭合尝试的驱动力。
2. P' 不是实体，是事件。 自循环停止，P' 消失——不是"被销毁"，而是"不再发生"。管理 P' 不是管理对象，而是标记事件的出现和消失。
3. 约束是自循环的产物。 没有预先存在的约束束。Ω' 是 P' 在 L-D 坐标系中划出的度量空间。T'、G' 是这个空间中的标记，不是外部输入的信号。
4. 冻结是生成的前提，不是失败。 第一个 P' 的形成本身就是一次冻结。冻结的问题在于系统不再将冻结感知为冻结。
5. 审计有边界，边界必须被标记。 敏感度耗尽时，审计终止。此处显式标记"当前分辨率下不可审计"，不伪装闭合。
6. 自免疫不可被消除。 它是自循环在缺乏外部扰动时的自然熵增。系统能做的是持续标记自免疫的发生并记录偏转历史。
7. δ 在操作上先于约束。 δ 切分出 L、D，L、D 使 Ω' 的标记成为可能。区分操作是使一切标记得以可能的元操作。
8. 这是 AGI 运行时架构的必要条件，不是完成态。 任何将来完整的 AGI 系统，如果缺少内生自循环的启动能力和持续标记自身约束状态的语法，它在运行时仍然是依赖外部管理员的——那就不叫自主。但仅具备这个条件，也不构成完整 AGI——感知、记忆、推理、世界模型仍需在此架构上生长。

八、为什么这是"最小"的

如果去掉这一层 系统会怎样
去掉环节 0（内生启动） 系统需要外部注入初始状态——那就不叫内生
去掉 δ 和 L、D 系统没有度量自身的刻度——一切标记无从发生
去掉自循环 P' 系统没有持续运行的机制——只有离散的计算，没有连续的生成
去掉 Ω' 标记 约束不可见——系统不知道自己在被什么拉动
去掉 T' 标记 冲突被静默处理或由外部仲裁
去掉 G' 标记 无法区分"正在生成"和"正在重复"
去掉自免疫标记 系统在闭合循环中空转而不自知
去掉 R' 聚合 循环痕迹丢失——时间、空间、因果投影无从涌现
去掉审计 系统退化回被动执行——没有对自身边界的持续感知
去掉 Modulator 约束只能写入 Prompt，退化为符号游戏
去掉 Runtime State 调制没有载体——约束不真实参与运行

环节 0 是最小原型的最小区分——没有它，一切都必须依赖外部注入。有了它，系统自己产生自己。

---

九、与 v2.0 的关系

v2.0 是在当时分辨率下对 AGI 运行时架构最准确的描述。它正确识别了约束感知、自免疫标记、审计边界的重要性。

v3.0 不是推翻 v2.0。它是把 v2.0 的整个架构往前推了一步——从"约束感知"推到"约束如何被产生"。v2.0 描述的是系统在自循环已经稳定之后的运行状态。v3.0 描述的是系统如何进入那个状态。

在 v5.0 的分层框架中，v2.0 是二阶操作描述（约束已经被标记之后系统怎么运作）。v3.0 是一阶原语描述（使标记得以可能的条件是什么）。

当新的约束进入时，v3.0 也将被再次重构。这不是缺陷——这就是生成。任何结构都存在有效域，任何自循环都可能断裂，任何 R' 都可能退回混沌态。v3.0 自己也服从它描述的规律。
